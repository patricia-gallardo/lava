The Floor is Lava

Trying to teach C++

Patricia Aas, NDC TechTown 2022

C++ is vast

Teaching an introduction to C++ requires us to select a subset of the language that will be the starting point for the rest of their learning.

Choosing a subset

To select such a subset we have to have some criteria.

Criteria

Has to be useful, a person should be able to write programs in C++ with it

Should be mostly harmless, sharp edges should be avoided if possible

Address the obvious, but keep it simple

No magic, no secrets

Sharp edges in plain sight need to be explained

Tooling to find bugs early have to be introduced from the beginning

What you teach has to be understood, minimize handwavy explanations

Like all programming languages C++ has a culture

But there is no definite list of what C++'s sharp edges are

We could try to suss out a sense of what the community feels

How can we sort C++ features after these rather vague criteria?

Useful

Mostly Harmless

Obvious

Take some C++ features

Sort on harmlessness

And sort on how advanced

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.on( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );
